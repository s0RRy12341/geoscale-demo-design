# Geoscale — מסמך אפיון עיצובי מלא לאינה

**תאריך:** 25.03.2026
**פרויקט:** עיצוב מחדש של מוצר Geoscale
**קישור לדמו:** https://geoscale-demo-design.vercel.app
**קוד מקור:** https://github.com/s0RRy12341/geoscale-demo-design

---

## מבוא

המסמך הזה מתאר את כל המסכים ב-Demo Design של Geoscale — פלטפורמה לניטור נוכחות מותגים במנועי AI (ChatGPT, Gemini).
הדמו בנוי ב-Next.js כ-clickable prototype עם דאטה סטטי. כל מה שמוצג כאן הוא **כיוון עיצובי** ולא מוצר סופי.

**חשוב:** כל העיצוב הוא RTL (עברית, ימין-לשמאל). הטקסטים בעברית, הכיוון הוא `dir="rtl"`.

## 1. שפה עיצובית (Design Language)

### צבעים — צבעי המותג המדויקים

| שם | קוד HEX | שימוש |
|---|---|---|
| **שחור** | `#000000` | כותרות ראשיות, טקסט ראשי |
| **שחור-כמעט** | `#141414` | לוגו wordmark, רקע כהה |

| **בורדר** | `#BFBFBF` | גבול כרטיסים, קווים מפרידים |
| **בורדר דק** | `#DDDDDD` | גבולות משניים, בדג'ים |
| **טקסט גוף** | `#333333` | טקסט תוכן רגיל |
| **טקסט מעומעם** | `#727272` | טקסט משני, תיאורים |
| **טקסט בהיר** | `#777777` | טקסט עזר |

- **אין גרדיאנטים** — בכלל
- **אין צלליות (box-shadow)** — בכלל

### טיפוגרפיה

| פונט | שימוש |

| **Inter** | UI ראשי — כפתורים, ניווט, מספרים, תוויות |

- כותרת עמוד: 26px, fontWeight 700

- מספרים גדולים (מטריקות): 36-40px, fontWeight 700

### מרכיבים בסיסיים

## 2. הדר (Header) — אחיד בכל הדפים

**מבנה:** CSS Grid עם 3 עמודות: `gridTemplateColumns: "1fr auto 1fr"`

```
┌─────────────────────────────────────────────────────┐
│  [כפתורים]           [ניווט]           [לוגו]      │
│  justifySelf: start   auto-center   justifySelf: end│
└─────────────────────────────────────────────────────┘
```

- **גובה:** 72px
- **רוחב מקסימלי:** 1300px, ממורכז עם `margin: 0 auto`
- **פדינג:** `0 24px`
- **רקע:** `rgba(255,255,255,0.96)` (שקוף מעט)
- **בורדר תחתון:** `1px solid #BFBFBF`
- **Position:** `sticky`, `top: 0`, `z-index: 50`

### עמודה ימנית (RTL = start): פעולות
- כפתור "סריקה חדשה" (שחור)
- אינדיקטור "מחובר" (נקודה ירוקה + טקסט)

### עמודה מרכזית: ניווט
- 2 לשוניות: **דשבורד** | **סריקות**
- הלשונייה הפעילה: `color: #000000`, `borderBottom: 2px solid #000000`
- לשונייה לא פעילה: `color: #727272`
- **אין** "מוצרים / שירותים" בניווט העליון (זה בתוך דף הסריקה)

### עמודה שמאלית (RTL = end): לוגו
- לוגו Geoscale (SVG — טבעת + wordmark)
- `justifySelf: end` כדי שיהיה בצד שמאל ויזואלית

### מבנה הדף (מלמעלה למטה):

- כותרת: "המותגים שלי" — `fontSize: 26px`, `fontWeight: 700`
- תת-כותרת: "X מותגים פעילים" — `fontSize: 14px`, `color: #727272`

#### 4.2 כרטיסי מותגים (BrandCard)
כל מותג הוא כרטיס שניתן להרחבה (expandable):

**מצב סגור:**
```
┌─────────────────────────────────────────────────────┐
│  [אייקון מותג]  שם המותג  |  domain.co.il          │

**מצב פתוח (expanded):** נפתח למטה עם:

1. **מטריקות גדולות** — 4 מספרים גדולים (fontSize: 36-40px):
   - סה"כ שאילתות

   - מוזכר ב-Gemini
   - ציון כולל (%)

2. **פעולות ממתינות** — קישורים לפעולות (צבע טורקיז):

   - "תוכנית פעולה מוכנה"
   - "הגדרת יעד חודשי"

3. **פעילות אחרונה** — רשימת שורות עם תאריכים:
   - "סריקה הושלמה — 37 שאילתות נבדקו"
   - "2 מתחרים חדשים זוהו"
   וכו'

4. **כפתורים:**
   - "צפה בסריקה" → מעביר ל-`/scan`

כשמחפשים: אנימציית pulse עדינה

## 5. דף 2: תוצאות סריקה ( /scan )

**URL:** `/scan`
**מטרה:** דשבורד מלא של תוצאות סריקה עבור מותג ספציפי (דוגמה: All4Horses)

### 5.1 כותרת מותג (Brand Header)
```
┌─────────────────────────────────────────────────────┐
│  [עיגול ציון 76%]   All4Horses                      │
│                      all4horses.co.il                │
│                      סריקה אחרונה: 15.03.2026        │
│                                   [סרוק שוב] כפתור  │
└─────────────────────────────────────────────────────┘
```
- Progress Ring (עיגול SVG) עם אחוז הציון
- שם מותג, דומיין, תאריך סריקה אחרונה
- כפתור "סרוק שוב" (שחור)

### 5.2 ארבע לשוניות (Tabs)
```
[סקירה] [שאילתות] [קהלים] [מוצרים / שירותים]
```
- לשונייה פעילה: `borderBottom: 2px solid #000000`
- **"מוצרים / שירותים" היא לשונייה רביעית בתוך דף הסריקה — לא בתפריט העליון!**
- זו פיצ'ר שמיוחד לכל מותג בנפרד

---

### TAB 1: סקירה (Overview)

#### 5.3 שורת מטריקות
4 כרטיסים בשורה אחת:

| מטריקה | דוגמה | הערות |
|---|---|---|
| סה"כ שאילתות | 37 | מספר גדול (text-4xl) |
| מוזכר ב-ChatGPT | 25 מתוך 37 | מספר גדול + "מתוך X" |
| מוזכר ב-Gemini | 28 מתוך 37 | מספר גדול + "מתוך X" |
| מתחרים | 4 | מספר גדול |

#### 5.4 השוואת מנועי AI
**שני כרטיסים זה לצד זה:**

```
┌────────────────────┐  ┌────────────────────┐
│ ChatGPT (GPT-4o)   │  │ Google Gemini       │
│ 68%                │  │ 76%                 │
│ [===== progress ===]│  │ [===== progress ===]│
│ 25 / 37 שאילתות    │  │ 28 / 37 שאילתות    │
└────────────────────┘  └────────────────────┘
```

- אייקון ChatGPT בצבע `#10A37F`
- אייקון Gemini בצבע `#4285F4`
- progress bar עם `#10A37F` (GPT) ו-`#4285F4` (Gemini)
- **אין** stacked bars, **אין** bar charts — רק שני כרטיסים נקיים

#### 5.5 שלבי מסע לקוח (Journey Stages)
**כרטיסים קומפקטיים בשורה אופקית** (לא progress bars!):

```
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│ 85%  │ │ 78%  │ │ 62%  │ │ 90%  │ │ 95%  │
│חשיפה │ │מחקר  │ │החלטה │ │תמיכה │ │מוניטין│
│7 שאי' │ │15שאי'│ │11שאי'│ │2 שאי'│ │2 שאי'│
└──────┘ └──────┘ └──────┘ └──────┘ └──────┘
```

- Grid: `gridTemplateColumns: repeat(5, 1fr)`, `gap: 12px`
- כל כרטיס: `padding: 16px`, `textAlign: center`
- אחוז גדול (22px, fontWeight 700)
- אם אחוז >= 80%: צבע `#10A37F`, אחרת `#000000`
- שם השלב (13px, fontWeight 600)
- מספר שאילתות (12px, color #727272)
- **שינוי מהמקור:** אלכסיי ביקש להחליף את ה-progress bars האופקיים (שתפסו מלא מקום וריקים מתוכן) לפורמט הקומפקטי הזה

#### 5.6 איתותים חיוביים ושליליים
שני כרטיסים זה לצד זה:

**חיוביים** (סימן V ירוק):
- רשימת bullet points עם סימון `#10A37F`
- דוגמה: "אזכור עקבי בשאילתות טיפול באמצעות סוסים"

**שליליים** (סימן X אפור):
- רשימת bullet points עם סימון `#727272`
- דוגמה: "חוסר אזכור בשאילתות מחיר ועלויות"

#### 5.7 Top 5 שאילתות
טבלה מינימלית עם 5 שורות הכי חשובות:
- מספר | טקסט שאילתה | פרסונה (בדג') | GPT (V/X) | Gemini (V/X)

#### 5.8 מתחרים
רשימת 4 מתחרים עם ציון:
- שם מתחרה + progress bar + אחוז

#### 5.9 חיבור SEO-GEO
כרטיס עם toggle לSEO ולGEO:
- טבלה: מילת מפתח | נפח חיפוש | קושי | שאילתות קשורות

### TAB 2: שאילתות (Queries)

#### 5.10 טבלת שאילתות מלאה
```
┌────────────────────────────────────────────────────────┐
│ # │ שאילתה              │ פרסונה │ שלב   │ GPT │ Gemini│
├────────────────────────────────────────────────────────┤
│ 1 │ רכיבה טיפולית...    │ [מאיה] │ [מחקר]│  V  │   V   │
│   └── [פתוח: תשובות מנועים + כפתור "הצג תשובה מלאה"]  │
├────────────────────────────────────────────────────────┤
│ 2 │ חוות סוסים...        │ [יוסי] │[חשיפה]│  V  │   V   │
└────────────────────────────────────────────────────────┘
```

**מנגנון פתיחה:**
1. לוחצים על שורה → נפתח panel מתחתיה עם רקע `#F9F9F9`
2. בתוך ה-panel: שני כרטיסים — ChatGPT ו-Gemini
3. כל כרטיס מציג snippet (קטע מקוצר מהתשובה)
4. **חדש: כפתור "הצג תשובה מלאה"** — לוחצים → הsnippet מתחלף לתשובה המלאה
5. כפתור "הסתר תשובה מלאה" — חוזר לsnippet

**פילטרים:**
- סטטוס: הכל | מוזכר | חסר | שלילי
- פרסונה: dropdown לסינון לפי פרסונה ספציפית

**שורת סיכום תחתונה:**
- "מציג X מתוך Y שאילתות"
- ספירות: מוזכר: X | חסר: X | שלילי: 0

## 6. דף 3: סריקה חדשה ( /new-scan )

**URL:** `/new-scan`
**מטרה:** תהליך 4 מסכים ליצירת סריקה חדשה

### מסך 1: הזנת מותג
```
┌─────────────────────────────────────────────────────┐
│                    [הדר]                             │
│                                                     │
│            לוגו Geoscale (גדול)                     │
│            "סריקת נוכחות AI למותג שלך"              │
│                                                     │
│            [דומיין: _____________]                   │
│            [שם מותג: ___________]                   │
│            [שפת סריקה: dropdown]                    │
│                                                     │
│            [   התחל סריקה   ]                       │
│                                                     │
│                    [פוטר]                            │
└─────────────────────────────────────────────────────┘
```

- לוגו גדול במרכז (רינג + wordmark)
- כותרת: "סריקת נוכחות AI למותג שלך"
- 3 שדות input (דומיין, שם מותג, שפה)
- כפתור "התחל סריקה" — שחור, רוחב מלא

### מסך 2: אנימציית ניתוח
- לוגו מסתובב במרכז עם אנימציית pulse
- שורת התקדמות (progress bar) עם shimmer
- טקסט מתחלף: "מנתח את הדומיין...", "סורק מנועי AI...", "מזהה פרסונות..."
- מספרים שמתעדכנים (שאילתות שנבדקו, מקורות שנסרקו)
- **אנימציות מוגדרות ב-globals.css** (pulse-ring, orbit, morph-circle, typing-cursor)

### מסך 3: תוצאות פרסונות
```
┌─────────────────────────────────────────────────────┐
│                    [הדר]                             │
│                                                     │
│     "זיהינו X קהלי יעד פוטנציאליים"               │
│     [sub-header: שם מותג | דומיין | שאילתות]        │
│                                                     │
│     ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│     │ פרסונה 1 │ │ פרסונה 2 │ │ פרסונה 3 │          │
│     └──────────┘ └──────────┘ └──────────┘          │
│     ┌──────────┐ ┌──────────┐                       │
│     │ פרסונה 4 │ │ פרסונה 5 │ [+ הצע פרסונה]       │
│     └──────────┘ └──────────┘                       │
│                                                     │
│     [   המשך לסריקה   ]                             │
│                                                     │
│                    [פוטר]                            │
└─────────────────────────────────────────────────────┘
```

- כותרת "זיהינו X קהלי יעד פוטנציאליים"
- כרטיסי פרסונות (grid 3 עמודות)
- כל כרטיס: שם, תפקיד, תגיות, checkbox לאישור
- כרטיס "+" להצעת פרסונה חדשה
- כפתור "המשך לסריקה"

### מסך 4: תהליך הסריקה (Process)
```
┌─────────────────────────────────────────────────────┐
│                    [הדר]                             │
│                                                     │
│     Step indicator: [1 ✓] → [2 ●] → [3] → [4]      │
│                                                     │
│     ┌───────────────────────────────────────────┐   │

## 8. הערות ושינויים מהמקור

### שינויים שבוצעו לאחר פידבק:

1. **הדר — יישור 3 עמודות:** הועבר מ-flexbox ל-CSS Grid עם `1fr auto 1fr` כדי ליצור "קווים דמיוניים" — הלוגו, הניווט והכפתורים תמיד מיושרים בדיוק בכל הדפים

2. **"מוצרים / שירותים" הוסרו מהניווט העליון:** זה פיצ'ר שמיוחד לכל מותג בנפרד, לא דף גלובלי. הועבר ללשונייה רביעית בתוך דף `/scan`

3. **שלבי מסע לקוח — שינוי פורמט:** Progress bars אופקיים הוחלפו בכרטיסים קומפקטיים (פידבק מאלכסיי — "תופס מלא מקום וריק מתוכן")

4. **השוואת מנועי AI — הסרת stacked bars:** הגרף שהראה "מוזכר/לא מוזכר" בבארים ויזואליים הוסר — נשארו רק שני כרטיסים נקיים עם אחוז + progress bar

5. **"הצג תשובה מלאה":** נוסף כפתור בכל snippet של ChatGPT/Gemini שמאפשר לראות את התשובה המלאה (לא רק קטע מקוצר)

6. **הדר ופוטר בכל מסכי סריקה חדשה:** נוספו לכל 4 המסכים של `/new-scan`
